Here is the context about the project to populate the report:

**System Overview:**
- Frontend: React 19 (Vite) + Tailwind CSS, deployed on Vercel.
- Backend: Node.js + Express, deployed on Render.
- Database: MongoDB Atlas.
- Real-time Telemetry: Unidirectional WebSocket push from server to client.
- The system exposes a unified API `/route` that hides vendor complexity. Third-party vendors are simulated with randomized latency, failures, and timeouts to test the routing engine.

**Architecture Flow:**
Client -> Express API -> Zod Validation -> Filter Ineligible Vendors -> Strategy Selector -> Failover Loop -> Simulated Vendor -> Circuit Breaker metrics update -> DB Audit Log -> WebSocket Push -> Response.

**Routing Decisions & Failover Logic:**
- The engine implements 8 strategies: Priority, Weighted, Round-Robin, Lowest Latency, Lowest Cost, Health-Based, Feature-Based, and Failover.
- Filtering: Before ranking, vendors are excluded if they are down, rate-limited, missing required features, above a latency threshold, or marked 'unhealthy' by the circuit breaker.
- Failover Loop: If the top-ranked vendor fails, times out, or errors, the system automatically tries the next-best vendor in the ranked list until success or exhaustion. A `failoverHistory` audit trail is returned in every response.
- Circuit Breaker: Automatically trips to 'Open' (unhealthy) if a vendor's error rate exceeds 50% over 5 requests, with a 'Half-Open' recovery probe after 60 seconds.
